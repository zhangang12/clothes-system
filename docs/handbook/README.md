## 核心业务规则速查

### 1. 状态机驱动
- **样衣**：PENDING → SAMPLING → SHIPPED → RETURNED → RECONCILED → DONE → ORDERED
- **报价**：DRAFT → QUOTED → ADJUSTING → ORDERED
- **订单**：DRAFT → CONFIRMED → CONTRACTED → PRODUCING → DONE
- **合同门户**：PUSHED → STAMPED → SHIPPING → RECONCILED → COMPLETED
- **对账**：DRAFT → PENDING → CONFIRMED → PAID
- **付款**：DRAFT → PENDING → APPROVED → PAID

### 2. 快照锁定机制
- 报价→合同：合同创建时锁定报价明细
- 供应商盖章：锁定合同材料单价和数量
- 发货批次：每批记录 `snapshot_unit_price`

### 3. 超付拦截（三层防护）
1. 对账单创建：发票号全局查重
2. 付款申请创建：悲观锁 + 累计申请额不超过对账总额
3. 分批付款登记：悲观锁 + 累计付款不超过应付额

### 4. 机密权限控制
- 客户机密行级权限：创建人/授权用户可见，管理员不限制
- 供应商门户：JWT + factory_id 绑定，仅见自己的合同

### 5. 编号生成规则
- 工厂：S001（前缀S + 3位序号）
- 客户：CM001（中间商）/ FE001（最终买家）
- 订单：PO-日期-序号
- 合同：HT-日期-序号（材料/加工）/ 补料-母合同号-序号（补料）

## 关键设计模式

1. **状态机驱动**：所有业务单据使用状态机管理生命周期
2. **快照锁定**：下游数据锁定上游快照，不受上游后续修改影响
3. **悲观锁串行化**：财务操作使用悲观锁防止并发超付
4. **机密行级权限**：客户数据按用户授权可见
5. **去重聚合**：错误日志使用指纹去重，避免日志膨胀
6. **双体系登录**：管理端和供应商门户使用不同的Token过期策略
