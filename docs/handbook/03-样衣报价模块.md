# 样衣管理 & 客户报价模块分析

## 模块组成

- **Sample** - 样衣管理（8个状态的状态机）
- **Quote** - 客户报价（含损金额计算、审批阈值）

| 状态 | 枚举值 | 中文标签 |
|------|--------|----------|
| 待派单 | PENDING | 待派单 |
| 打样中 | SAMPLING | 打样中 |

| 已寄回 | RETURNED | 已寄回 |
| 已对账 | RECONCILED | 已对账 |
| 已完成 | DONE | 已完成 |
| 已成单 | ORDERED | 已成单 |
| 已废弃 | ABANDONED | 已废弃 |

### 状态转换图

```
创建 ──→ PENDING（待派单）
             │
             ├─ pushPatternmaker() ──→ SAMPLING（打样中）
             │   前置: status ∈ {PENDING, SAMPLING}
             │
             ├─ remove() ──→ 软删除
             │   前置: status === PENDING && 无报价引用
             │
             └─ abandon() ──→ ABANDONED（已废弃）
             
SAMPLING（打样中）
  │
  ├─ markShipped() ──→ SHIPPED（已寄出）
  │
  ├─ patternmakerSave(returnNo) ──→ RETURNED（已寄回）
  │
  ├─ patternmakerSave(shipRounds) ──→ RECONCILED（已对账）
  │   公式: labor_amount = piece_count × labor_unit_price
  │
  └─ abandon() ──→ ABANDONED

SHIPPED（已寄出）
  │
  ├─ patternmakerSave(returnNo) ──→ RETURNED
  └─ patternmakerSave(shipRounds) ──→ RECONCILED

RETURNED（已寄回）
  │
  └─ patternmakerSave(shipRunks) ──→ RECONCILED

RECONCILED（已对账）
  │
  └─ complete() ──→ DONE（已完成）

DONE（已完成）
  │
  └─ complete() ──→ DONE（保持）

ORDERED（已成单）── 终态，由外部触发
  │
  └─ trigger: quoteService.toContract()
```

## 三、报价计算公式

### 核心公式链

```
含损金额 (loss_amount)
  = 人民币单价 (rmb_price) × 报价耗用 (quote_usage) × (1 + 损耗% / 100)
  默认损耗 = 3%
  代码位置: quote.service.ts:27-29 calcLossAmount()

美金单价 (usd_price)
  = 人民币单价 (rmb_price) / 汇率 (exchange_rate)
  代码位置: quote.service.ts:57

费用合计
  = Σ (费用明细.人民币单价 × 费用明细.报价耗用)
  默认费用行: 加工费/线/包装/样衣费/测试费/运费（6行）

报价人民币总价 (rmb_total)
  = (含损金额合计 + 费用合计) × (1 + 利润率% / 100)
  代码位置: quote.service.ts:78

报价美元总价 (usd_total)
  = 报价人民币总价 / 汇率
```

### 报价状态机（4个状态）

| 状态 | 枚举值 | 转换 |
|------|--------|------|
| 草稿 | DRAFT | 创建时默认 |
| 已报价 | QUOTED | 草稿/客户调整 → submit |
| 客户调整 | ADJUSTING | 已报价 → adjust |
| 已成单 | ORDERED | 已报价/客户调整 → toContract |

### 审批阈值机制

- 超阈值条件：
  1. `rmb_total > 阈值`（阈值存 sys_config `approval.quote.threshold`）
  2. `利润率 < 最低下限`（`quote.min_profit_rate`）
- 审批流程：NONE → PENDING（自动）→ APPROVED（主管审批）

## 四、从样衣导入报价的数据流

### 数据映射规则

```
样衣材料表 (SampleMaterial)           →  报价明细 (QuotationItem)
──────────────────────────────       ────────────────────────────
sample_material.part               →  quotation_item.part (部位)
sample_material.item_name          →  quotation_item.item_name (品名)
sample_material.width              →  quotation_item.width (门幅)
sample_material.colors             →  quotation_item.color (颜色)
sample_material.supplier_name      →  quotation_item.supplier (供应商)
sample_material.actual_usage       →  quotation_item.quote_usage (报价耗用)
(无映射)                           →  quotation_item.loss_rate = 3 (默认损耗)
```

### 样衣材料修改 → 同步未成单报价

触发时机：
1. 业务编辑样衣材料
2. 版师保存实际耗用

同步策略（按品名匹配）：
- 匹配到已有报价行 → 保留人民币单价/损耗率/单位/备注（议价不动）
- 耗用/颜色/供应商等字段从样衣刷新
- 已成单 (ORDERED) 报价不动

## 七、模块间依赖关系

```
SampleModule（样衣管理）
  imports → QuoteModule（样衣材料修改同步未成单报价）
  imports → CustomerModule（客户资料/机密遮蔽）

QuoteModule（客户报价）
  imports → CustomerModule（客户资料/机密行过滤）
  imports → OrderModule（转销售合同自动建订单）

依赖方向:
  Sample → Quote（同步报价）     单向
  Quote → Order（生成订单）      单向
  Quote → Sample（读取材料数据）  只读
  Sample ← Quote（toContract 置已成单）
```
