# I9 服装制造管理系统 - 系统架构概览

## 一、项目概述

面向服装外贸/制造企业的全流程数字化协同系统，覆盖 **样衣 → 报价 → 订单 → 合同 → 供应商门户 → 对账 → 付款 → 结算** 的完整业务闭环。

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **Monorepo** | pnpm workspace | `packages/{types,api,web,portal}` |
| **后端** | NestJS 10 + TypeORM + MySQL 8 + Redis 7 | JWT 鉴权、角色守卫、全局校验/异常/响应拦截、限流 |
| **PC 管理端** | Vue 3 + Element Plus | 靛蓝 #1E3A5F + 亚麻米 #F5EDDC + 铁锈橙 #D17A40 |
| **供应商门户** | Vue 3 + Vant | H5 移动端，DATEX 品牌色(墨绿#2E8B78) |
| **共享契约** | TypeScript | 全模块 TS 类型/枚举 |

## 四、模块注册 (`app.module.ts`)

### 全局配置
- `ConfigModule.forRoot({ isGlobal: true })` - 环境变量
- `ThrottlerModule` - 限流（登录 10次/min，其余 300次/min）
- `TypeOrmModule.forRootAsync` - 数据库连接

### 公共服务
- `NumberingModule` - 全局编号生成（Redis+DB双写兜底）

## 六、关键设计模式

### 1. 快照锁定机制
- **报价→合同**：合同创建时锁定报价明细
- **供应商盖章**：锁定合同材料单价和数量
- **发货批次**：每批记录 `snapshot_unit_price`，对账用发货时价格

### 2. 状态机驱动
- 样衣：8个状态（PENDING → SAMPLING → SHIPPED → RETURNED → RECONCILED → DONE → ORDERED）
- 报价：4个状态（DRAFT → QUOTED → ADJUSTING → ORDERED）
- 订单：5个状态（DRAFT → CONFIRMED → CONTRACTED → PRODUCING → DONE）
- 合同门户：5个状态（PUSHED → STAMPED → SHIPPING → RECONCILED → COMPLETED）

### 3. 超付拦截（三层防护）
1. 对账单创建：发票号全局查重
2. 付款申请创建：悲观锁 + 累计申请额不超过对账总额
3. 分批付款登记：悲观锁 + 累计付款不超过应付额

### 4. 机密权限控制
- 客户机密行级权限：创建人/授权用户可见，管理员不限制
- 供应商门户：JWT + factory_id 绑定，仅见自己的合同

## 八、开发规范

- **编号生成**：使用 `NumberingService`（Redis+DB双写兜底）
- **逻辑删除**：`deleted` 字段，不物理删除
- **DTO 校验**：`class-validator` + `WhitelistInterceptor`
- **角色守卫**：`@Roles()` + `RolesGuard`
- **变更日志**：`ChangeLogService` 记录关键操作
