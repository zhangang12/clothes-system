## 一、路由配置

```
/portal/login          → LoginView       (meta.public, 无需登录)
/portal                → PortalLayout    (meta.requiresAuth)
  ├── contracts        → ContractListView   (默认重定向)
  ├── contracts/:id    → ContractDetailView (meta.ownNav: 自带返回导航)
  └── mine             → MineView
/:pathMatch(.*)        → 重定向 /portal/login
```

**守卫机制**：`beforeEach` 检查 `localStorage.getItem('portal_token')`，未登录重定向登录页

## 二、四步流程——业务规则与状态转换

### 状态枚举

```
DRAFT → PUSHED → STAMPED → SHIPPING → RECONCILED → COMPLETED
```

门户可见状态仅限 `PUSHED / STAMPED / SHIPPING / RECONCILED / COMPLETED`（DRAFT 不暴露给供应商）

### 四步详情

| 步骤 | 状态转换 | 触发操作 | 前置条件 | 关键业务规则 |
|------|----------|----------|----------|-------------|
| **1. 盖章** | PUSHED → STAMPED | `PATCH /portal/contracts/:id/stamp` | 必须勾选"同意条款" | 支持纸质盖章照片或电子章；**盖章后锁定快照** |
| **2. 发货** | STAMPED → SHIPPING | `PATCH /portal/contracts/:id/ship` | STAMPED 或 SHIPPING 状态 | 支持整单或逐物料行填写实发数；可选收货地址 |
| **3. 对账** | SHIPPING → RECONCILED | `PATCH /portal/contracts/:id/reconcile` | SHIPPING 状态；批次须已审批 | 自动按快照单价×数量算金额；生成对账单推业务复核 |
| **4. 开票** | RECONCILED → COMPLETED | `PATCH /portal/contracts/:id/invoice` | RECONCILED 状态 | 发票号全局查重；发票金额须与对账金额差额 ≤¥0.01 |

### 辅助操作

| 操作 | API | 规则 |
|------|-----|------|
| 发货完成 | `PATCH /portal/contracts/:id/ship-done` | 仅 SHIPPING/RECONCILED 可用 |
| 撤回发货批次 | `DELETE /portal/contracts/:id/shipments/:sid` | 未被对账占用才可撤 |
| 合并发货 | `POST /portal/contracts/merge-ship` | 至少勾选 2 张同供应商合同 |

## 三、权限隔离机制（三重隔离）

### 1. 路由层

无 `portal_token` 直接重定向登录页

### 2. API Guard 双重校验

- `JwtAuthGuard`：验证 JWT Token 有效性，提取 `req.user`
- `SupplierGuard`：检查 `req.user.type === 'supplier'`，非供应商直接 `ForbiddenException`

### 3. Service 层 factory_id 绑定（最关键）

- 合同列表查询：`WHERE factory_id = :factoryId`
- 合同详情查询：`WHERE id = :id AND factory_id = :factoryId`
- 所有操作（盖章/发货/对账/开票/撤回）均在 `findOne` 时传入 `factoryId`

**结论**：供应商只能看到自己工厂的合同，无法越权访问其他供应商的数据

## 四、快照锁定机制

### 盖章时锁定

盖章瞬间创建 `snapshot_json`，锁定以下字段：

```json
{
  "contract_no", "type", "factory_id", "order_id",
  "total_amount", "currency", "deposit_ratio", "mid_ratio",
  "final_ratio", "account_period_days",
  "materials": [{ "item_name", "spec", "unit", "unit_price", "qty", "amount" }],
  "stamped_at": "ISO时间戳"
}
```

### 锁定规则

- **触发时机**：仅在 PUSHED → STAMPED 时写入
- **锁定内容**：合同金额、付款比例、全部材料明细（品名/规格/单价/数量/小计）
- **发货时锁价**：每批发货记录 `snapshot_unit_price`，确保对账用的是发货时价格
- **撤销重推**：内部修改合同重新推送时，`revised=1` 使前端显示警告条

## 五、API 接口清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/portal/contracts` | 合同列表(分页+按状态筛选) |
| GET | `/portal/contracts/:id` | 合同详情 |
| PATCH | `/portal/contracts/:id/stamp` | 供应商盖章(锁定快照) |
| POST | `/portal/contracts/merge-ship` | 跨合同合并发货 |
| PATCH | `/portal/contracts/:id/ship` | 确认出货(创建发货批次) |
| PATCH | `/portal/contracts/:id/ship-done` | 宣布发货完成 |
| DELETE | `/portal/contracts/:id/shipments/:sid` | 撤回发货批次 |
| PATCH | `/portal/contracts/:id/reconcile` | 我要对账 |
| PATCH | `/portal/contracts/:id/invoice` | 上传发票 |
| POST | `/auth/portal/login` | 供应商登录 |
| PATCH | `/auth/change-password` | 修改密码 |
