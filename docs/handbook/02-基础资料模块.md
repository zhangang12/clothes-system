# 基础资料模块分析

## 模块组成

- **Company** - 本司主体（合同甲方/PDF抬头/收款信息）
- **Factory** - 工厂/厂商资料（35字段+联系人子表）
- **Customer** - 客户资料（47字段+联系人/开户银行/快件三子表）
- **Dict** - 通用字典（下拉自填累积）

---

## 一、Company（本司主体）

### 表结构 `company_profile`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| name | varchar(100) | 公司全称（PDF抬头/合同甲方） |
| short_name | varchar(50) | 简称 |
| address | varchar(200) | 地址 |
| phone | varchar(30) | 电话 |
| tax_no | varchar(30) | 税号 |
| bank_name | varchar(100) | 开户行 |
| bank_account | varchar(40) | 银行账号 |
| legal_rep | varchar(50) | 法定代表人 |
| logo_url | varchar(500) | 抬头Logo |
| seal_url | varchar(500) | 电子章图片 |
| is_default | tinyint | 1=默认主体 |
| deleted | tinyint | 逻辑删除 |

### 核心业务规则

1. **默认主体唯一性**：`makeSoneDefault()` - 置某条为默认时，先将所有其他记录 `is_default=0`
2. **默认主体供外部取数**：`getDefault()` 先查 `is_default=1`，无则返回最早一条
3. **逻辑删除**：`removed=1`，查询均过滤 `deleted=0`

### API 接口

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| GET | `/company-profiles` | ADMIN/BUSINESS/FINANCE/SUPERVISOR | 列表 |
| GET | `/company-profiles/default` | 同上 | 默认主体 |
| GET | `/company-profiles/:id` | 同上 | 详情 |
| POST | `/company-profiles` | ADMIN | 新增 |
| PUT | `/company-profiles/:id` | ADMIN | 更新 |
| PATCH | `/company-profiles/:id/set-default` | ADMIN | 设为默认 |
| DELETE | `/company-profiles/:id` | ADMIN | 删除 |

## 二、Factory（工厂/厂商资料）

### 主表 `factory`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| factory_no | varchar(20) UNIQUE | S001 格式，全局递增 |
| type | enum | 主身份（FABRIC/PROCESS等） |
| extra_types | varchar(100) | 附加身份（逗号分隔） |
| seal_url | varchar(500) | 电子章图片 |
| can_invoice | tinyint | 能否开票 |
| name | varchar(100) | 厂商名称 |
| contact_name | varchar(50) | 主联系人（自动回填） |
| contact_phone | varchar(30) | 主联系人电话（自动回填） |
| grade | varchar(10) | 信用等级 A/B/C |
| status | tinyint | 1=启用 0=停用 |

### 子表 `factory_contact`

| 字段 | 说明 |
|------|------|
| factory_id | FK关联工厂 |
| sort_order | 排序 |
| name/department/title | 联系人信息 |
| phone/mobile/email | 联系方式 |

### 核心业务规则

1. **编号生成**：`NumberingService.nextGlobal(NUM_PREFIX.FACTORY)` - 前缀 `S`，全局递增
2. **名称唯一性**：`assertNameUnique()` 检查同名未删除记录
3. **联系人非空**：保存时至少一个联系人
4. **主联系人自动回填**：取联系人子表首行的 name/phone 回填主表
5. **双身份设计**：`type`(主) + `extra_types`(附加逗号分隔)
6. **停用检查**：有未完成合同则阻止停用
7. **删除引用检查**：被 contract/order/reconciliation/payment/sample 引用则阻止删除

### API 接口

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| POST | `/factories` | ADMIN/BUSINESS | 创建 |
| POST | `/factories/import` | ADMIN/BUSINESS | 批量导入 |
| GET | `/factories` | 全员 | 分页列表 |
| GET | `/factories/select` | 全员 | 下拉列表(仅启用) |
| GET | `/factories/:id` | 全员 | 详情 |
| PUT | `/factories/:id` | ADMIN/BUSINESS | 更新 |
| PATCH | `/factories/:id/status` | ADMIN | 启用/停用 |
| DELETE | `/factories/:id` | ADMIN | 逻辑删除 |

## 三、Customer（客户资料）

### 主表 `customer`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| customer_no | varchar(20) UNIQUE | CM001/FE001 格式 |
| name | varchar(100) | 客户名称 |
| type | enum | MIDDLEMAN/BUYER |
| related_middleman | varchar(200) | 关联中间商ID串（BUYER必填） |
| currency | varchar(5) | 主结算币种（默认USD） |
| commission_rate | decimal(5,2) | 中间商默认佣金率% |
| grade | enum(A/B/C) | 信用等级 |
| status | tinyint | 1=启用 0=停用 |

### 子表

| 子表 | 字段数 | 说明 |
|------|--------|------|
| customer_contact | 10 | 联系人信息 |
| customer_bank | 7 | 开户银行 |
| customer_express | 4 | 快件公司 |
| customer_grant | - | 机密授权表 |

### 核心业务规则

1. **编号按类型分前缀**：中间商 `CM-`，最终买家 `FE-`
2. **类型联动**：type=BUYER 时必须选关联中间商
3. **机密行级权限（核心机制）**：
   - 创建人天然可见+可编辑
   - 管理员不限制
   - 非授权用户看到 404（防探测）
   - 支持授权有效期，过期自动失效
4. **改名实时同步**：客户改名 → 同步草稿/已报价的 quotation/sample_garment/order_main 名称快照
5. **停用检查**：有未完成订单则阻止停用

### API 接口

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| POST | `/customers` | ADMIN/BUSINESS | 创建 |
| POST | `/customers/import` | ADMIN/BUSINESS | 批量导入 |
| GET | `/customers` | 全员 | 分页列表(机密过滤) |
| GET | `/customers/select` | 全员 | 下拉(机密过滤) |
| POST | `/customers/grants` | ADMIN | 批量授权 |
| GET | `/customers/:id` | 全员 | 详情(机密检查) |
| PUT | `/customers/:id` | ADMIN/BUSINESS | 更新(权限检查) |
| PATCH | `/customers/:id/status` | ADMIN | 启用/停用 |
| DELETE | `/customers/:id` | ADMIN | 逻辑删除 |

## 四、Dict（通用字典）

### 表结构 `sys_dict`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| type | varchar(40) | 字典类别 |
| label | varchar(100) | 显示值 |
| value | varchar(100) | 附加值（如币种汇率） |
| sort | int | 排序 |
| status | tinyint | 1=启用 0=停用 |

### 核心业务规则

1. **自填累积**：下拉允许自填新值，新值自动落库累积
2. **幂等创建**：同 type+label 已存在则恢复 status=1 并返回，不报错
3. **停用而非删除**：删除操作实为 `status=0`

### 支持的字典类别

consignee, destination, color, size, composition, trade_country, price_terms, settlement_method, currency, department, title, customer_source, cooperation_level, fee_item

```
┌─────────────────────────────────────────────────────────────┐
│                        Dict (字典)                          │

               │ 被引用（下拉自填累积）
               ▼
┌──────────────────────────────┐    ┌──────────────────────────┐
│        Company (本司主体)      │    │     Factory (工厂)        │
│  被 contract/invoice 取数     │◄───│  被 sample_material 引用   │
│  合同甲方/PDF抬头/收款        │    │  被 contract 引用          │
│  Module 导出供外部使用        │    │  被 order_main 引用        │
└──────────────────────────────┘    │  被 reconciliation 引用    │
                                    │  被 payment_request 引用   │
                                    │  被 prepayment 引用        │
                                    │  门户账号→supplier_account  │
                                    └─────────┬────────────────┘
                                              │
                    ┌─────────────────────────┐│
                    │    Customer (客户)        ││
                    │  type=BUYER 需关联中间商   ││
                    │  被 sample_garment 引用    ││
                    │  被 quotation 引用         ││
                    │  被 order_main 引用        ││
                    │  机密权限→customer_grant   ││
                    │  改名→同步快照到            ││
                    │    quotation/sample/order  ││
                    │  快件公司→Factory(FORWARDER)│┘
                    └─────────────────────────┘
```
