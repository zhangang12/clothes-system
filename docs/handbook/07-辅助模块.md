# 辅助模块与基础设施分析

## 模块组成

- **Auth** - 认证授权（双体系登录）
- **Upload** - 文件上传（安全校验）
- **ErrorLog** - 错误日志（自动归档）
- **Feedback** - 反馈管理
- **Ops** - 运维监控（ECDSA签名）
- **Stats** - 统计分析（四大维度）
- **Infra** - 基础设施（部署/脚本）

---

## 一、认证授权 (Auth)

### 双体系登录

| 机制 | 管理端 | 供应商门户 |
|------|--------|------------|
| 密码哈希 | bcryptjs, salt=10 | 同 |
| 令牌签发 | @nestjs/jwt | 同 |
| Token 过期 | 8h | 30d |
| Payload | sub/username/role/type:'admin' | + factory_id/type:'supplier' |
| 登录限流 | 10次/分钟 | 同 |

### 角色守卫

- `JwtAuthGuard`：包装 `@nestjs/passport` 的 `AuthGuard('jwt')`
- `RolesGuard`：读取 `@Roles()` 装饰器元数据，校验 `req.user.role`
- **硬隔离**：`user.type === 'supplier'` 时直接拒绝管理端所有接口

### 用户模型

| 表 | 用途 |
|----|------|
| sys_user | 管理端用户（id/username/password/real_name/role/status） |
| supplier_account | 供应商账号（id/account/password/factory_id/status/last_login_at） |

### 安全兜底

不允许停用最后一个启用的管理员

## 二、文件上传 (Upload)

### 存储架构

- 存储路径：`/data/uploads/`（`UPLOAD_ROOT` 环境变量配置）
- 按日期分子目录：`uploads/{misc|private}/{YYYY}/{MM}/{uuid}.{ext}`
- 文件名：`uuid v4` 重命名，不保留原始文件名
- 敏感附件：通过 `?sensitive=1` 存入 `private/` 子目录

### 文件类型限制（两层校验）

| 层 | 机制 |
|----|------|
| 第一层：MIME 白名单 | 允许 image/jpeg/png/webp、application/pdf、xlsx/xls、octet-stream/zip |
| 第二层：Magic Bytes 内容校验 | 落盘后读前 16 字节判定真实类型，不支持则删除并 400 |
| 扩展名对齐 | 若真实类型与扩展名不匹配，强制改名 |

### 大小限制

- Multer `fileSize: 20 * 1024 * 1024`（20MB）
- nginx `client_max_body_size 25M`

### 安全措施

- 敏感附件：HMAC-SHA256 签名，5 分钟有效
- 路径穿越防护：`..` 被清除 + `startsWith(uploadRoot)` 校验
- 响应安全映射：非图片/PDF 一律 `application/octet-stream`，`X-Content-Type-Options: nosniff`

## 三、错误日志 (ErrorLog)

### 自动归档机制

- 全局异常过滤器 `HttpExceptionFilter` 通过 `APP_FILTER` 注册
- 所有 **>=500** 异常自动写入 `error_log` 表
- **去重聚合**：`fingerprint = SHA1(method + 归一化路径 + errorType + message前120字)`，同类报错 count+1

### 数据模型

| 字段 | 用途 |
|------|------|
| fingerprint | 去重指纹（唯一索引） |
| method/path/status_code | 请求信息 |
| error_type/message/stack | 错误信息 |
| req_input/resp_output | 操作上下文（密码字段脱敏） |
| user_id/username/ip | 触发用户信息 |
| count | 累计次数 |
| status | OPEN/HANDLED |
| first_seen/last_seen | 首次/末次出现时间 |

### 密码脱敏

`body` 中匹配 `pass/pwd/secret/token` 键名的字段替换为 `***`

## 四、反馈管理 (Feedback)

### 数据模型

| 字段 | 用途 |
|------|------|
| user_id/username | 提交用户 |
| content | 问题描述（1-2000字） |
| images | 图片 URL JSON 数组 |
| page_url | 提交页面上下文 |
| status | PENDING / HANDLED |
| reply/reply_at | 管理员回复内容+时间 |
| reply_read | 是否已读（触发前端红点） |

### 功能流程

- 提交：任意内部登录用户可提交
- 管理：管理员查看列表/标记处理/附回复/导出 HTML
- 自助：查看我的反馈、未读回复数（红点）、标记已读

## 五、运维监控 (Ops)

### 双层鉴权架构

| 层 | 机制 |
|----|------|
| 服务端验签 | ECDSA P-256 公钥验签 |
| 客户端签名 | 离线监控页内嵌 JWK 私钥，每次请求签名 |
| 防重放 | 时间戳 SKEW_SEC = 180s |

### 监控数据

- `systemTime`：北京时间
- `errors`：error_log 表 OPEN 状态计数
- `feedback`：feedback 表 PENDING 状态计数
- `backup`：读日志解析最近备份时间/大小
- `watch`：读日志获取最近巡检结果
- `loop`：读 JSON 获取智能巡检心跳状态

### 离线监控页

- 单文件 HTML，60 秒自动刷新
- 断网回退 localStorage 缓存
- 展示 3 项定时作业：每日备份 / 30 分钟轻量巡检 / 智能巡检+自愈

## 六、统计分析 (Stats)

### 四大分析维度

| 维度 | 端点 | 数据源 |
|------|------|--------|
| 转化漏斗 | GET /stats/funnel | sample + quotation + order |
| 订单统计 | GET /stats/orders?dimension= | order_main |
| 成单率 | GET /stats/win-rate?dimension= | quotation |
| 利润汇总 | GET /stats/profit?dimension= | settlement |

### 转化漏斗

样衣建档 → 客户报价 → 销售成单
计算三级转化率：样衣->报价率、报价->成单率、总体成单率

### 利润分析

- 按款号/月份/客户分组
- 含 settleAmount/grossProfit/grossMargin%/netProfit
- **亏损预警**：`netProfit < 0` 时 `loss: true`，结果按净利润升序

## 七、部署架构

### 单机 All-in-One 部署

```
┌─────────────────────────────────────────┐
│                ECS 单机                   │
│                                          │
│  nginx (port 80)                         │
│    /api/  ──proxy_pass──>  :3000 (Node)  │
│    /      ──static──>  /var/www/web      │
│    /portal/ ──static──> /var/www/portal  │
│                                          │
│  systemd: i9-api (Node.js)              │
│    以 i9app 用户运行                      │
│    EnvironmentFile=.env.production       │
│    MemoryMax=1G, Restart=always          │
│                                          │
│  Docker (仅数据库，绑 127.0.0.1):        │
│    i9_mysql: MySQL 8.0                   │
│    i9_redis: Redis 7-alpine              │
│    数据卷: mysql_data / redis_data        │
│                                          │
│  文件存储: /data/uploads/                │
│  备份存储: /data/backups/                │
└─────────────────────────────────────────┘
```

### Docker Compose

- MySQL 8.0：utf8mb4，innodb-buffer-pool-size=256M，max-connections=200
- Redis 7：requirepass + appendonly yes
- 两者仅绑定 127.0.0.1，不暴露外网

### systemd

- Type=simple，User=i9app
- 安全加固：NoNewPrivileges、PrivateTmp、ProtectSystem=full
- 资源限制：LimitNOFILE=65536、MemoryMax=1G

### Nginx

- API 反代：`/api/ -> http://127.0.0.1:3000`，proxy_read_timeout 30s
- 文件访问：`/internal/files/` 为 internal 位置（X-Accel-Redirect）
- SPA：管理后台 `/` -> `/var/www/web/`，供应商 `/portal/` -> `/var/www/portal/`
- HTTPS 块已预留模板

### CI/CD

- CI：lint + typecheck + 后端 unit+e2e + 前端 test + 构建检查
- CD：push main -> SSH -> deploy.sh -> health.sh
- 部署幂等：git pull -> 构建 -> 备份 -> hotfix-schema.sql -> 重启 API -> 健康检查 -> nginx reload
