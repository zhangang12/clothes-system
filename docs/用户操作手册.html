00系统全景与数据流

一条端到端业务流：样衣 → 报价 → 订单 → 合同 → 供应商门户四步 → 对账 → 付款 → 结算 → 报表
③样衣 S- ──导入材料──▶ ④客户报价 Q- ──一键导入──▶ ⑤订单 O-（尺码PO矩阵→大货总数→用料核算） │ ┌─ 生成材料合同（按供应商拆多张，分色/分码按矩阵出行） └─ 生成加工合同（数量=大货数） ⑤′合同 HT- │ 推送（超阈值先主管审批） ▼ ⑥供应商门户 H5：🔖盖章 → 📦发货(FH-·多批·业务逐批审批) → 🧾我要对账(勾已审批批次→DZ-) → 💵开票(发票金额须=对账金额) │ │ 业务复核确认(二级审批) ◀─────────┘ 自动生成付款申请 PR-（待审批） ▼ ▼ ⑦对账 DZ-（合同/无合同/工时三类） ──▶ ⑧付款 PR-（预付冲抵·分批付款·账期到期日） ──▶ ⑨结算 JS-（按款号聚合成本·退税·净利） │ ⑩报表：漏斗 / 成单率 / 利润(亏损预警)

关键联动（系统自动，无需人工）

  • 报价「转销售合同」→ 关联样衣自动置 已成单；订单从报价导入自动带客户/明细快照。
  • 生成合同后订单自动 已下单→已生成合同；对账确认后合同→已对账、订单（生产中时）→已完成。
  • 门户开票成功 → 自动生成付款申请（直接进入财务审批台，带账期/到期日）。
  • 付款申请付清 → 关联对账单自动转 已付款。
  • 系统运行故障自动记录进「系统报错」，无需人工上报。

01登录 · 角色 · 权限

管理端（Web）与供应商门户（H5）双入口、双账号体系

操作指引

  1. 内部员工：浏览器打开管理端地址 → 输入用户名/密码登录。密码错误会在登录页直接提示。
  2. 供应商：手机打开门户 H5 地址 → 用工厂档案里开通的门户账号登录，只能看到推送给本厂的合同。

六个内部角色 + 权限要点

角色定位关键限制
ADMIN 管理员全权限独有：删除类操作、客户机密批量授权、反馈管理、系统报错、本司主体维护、用户清单
BUSINESS 业务建单主力可建/编辑 客户·样衣·报价·订单·合同·对账；受客户机密行级限制
FINANCE 财务钱的动作付款审批/分批付款登记/结算；报价可见
SUPERVISOR 主管审批节点超阈值报价/订单/合同审批、对账复核确认/退回、发货批次审批
PATTERNMAKER 版师打样执行不可见：客户报价、报表统计、本司主体菜单；合同/订单中供应商成本、对客单价被隐藏（显示「—」）
SAMPLE_MAKER 打样同版师同上脱敏与菜单限制
🔒 客户机密是独立于角色的行级控制：即使是业务/财务，看不到未授权的客户及其报价（详见 §2 客户资料）。管理员不受限。

02基础资料：工厂 / 客户 / 本司主体

全系统取数源头——报价选客户、合同选工厂、PDF 抬头取本司主体，都从这里来

2.1 工厂资料（供应商/加工厂档案）

  1. 基础资料 → 工厂资料 → 新建：填名称、主身份（面料/辅料供应商、委外加工商、货代、测试、出口、其他 7 类）、可开票、联系人/电话、地址、银行、税号等。编号自动 S001 递增。
  2. 双身份工厂（既供料又加工）：主身份选一类，附加身份补另一类——合同选甲方时按身份过滤，双身份两边都能被选到。
  3. CSV 批量导入：下载模板 → 填好上传 → 逐行校验入库，错误行单独提示。
  4. 要走门户协同的工厂：在工厂档案开通门户账号（推送合同前必须开通，否则推送被拦截并提示）。

业务分支与规则

分支系统行为
删除被引用的工厂拦截——被 合同/订单/对账/付款申请/预付款/样衣材料 任一引用即不可删，提示引用来源
停用工厂所有单据下拉不再出现；存量单据不受影响
合同选甲方过滤材料合同只列 面料/辅料/其他；加工合同只列 委外加工商/其他（含附加身份）

2.2 客户资料（机密单据 🔒）

  1. 新建：选类型——中间商（编号 CM001）或最终买家（编号 FE001，必须关联中间商，下拉只列中间商）。
  2. 联系人子表不能为空（保存时校验）；客户名称全局唯一；银行子表开户名自动=客户名称；另有快递账号子表。
  3. 保存成功 → 系统自动为创建人登记机密权限（查看+修改）。

🔒 机密授权（谁能看到这个客户）

场景操作 / 系统行为
默认可见范围创建人 + 被授权人 + 管理员。其他人：客户列表看不到这行、打开详情也只提示「不存在」、所有单据的客户下拉里也没有、该客户名下的报价单同样整单隐藏
批量授权（仅管理员）客户列表勾选多条 → 工具栏「批量授权机密权限」→ 勾选用户（多选）→ 选权限级别 仅查看 / 查看+修改 → 确认。重复授权自动幂等升降级
查看/撤销单客户授权行操作「授权」→ 弹出该客户的授权清单（用户/角色/级别）→ 可逐条撤销
仅查看者试图修改被拒并提示「仅有查看权限，无修改权限」
非管理员调授权系统直接拒绝

2.3 本司主体（多抬头）

管理员维护本司公司主体（全称/地址/税号/开户行/法人/Logo），可设默认主体。取数方：合同乙方/委托方下拉、合同与报价 PDF 抬头、（未来）开票抬头。

03样衣管理

业务建档 → 推版师打样 → 寄出/寄回 → 工时对账 → 完成/成单 · 编号 S-YYYYMMDD-序号

标准流程

  1. 业务新建样衣：款号、品类、中间商、材料明细等 → 状态 待派单。
  2. 点推送版师（可同时填材料寄出单号）→ 打样中。
  3. 版师视图保存：实际耗用、拉链长度、寄回单号、件数、工时单价（此步允许在 打样中/已寄出/已寄回/已对账 状态下补录）。
  4. 标记已寄出 → 已寄出；版师填寄回单号后 → 已寄回。
  5. 件数+工时单价齐了 → 走工时对账（见分支）→ 已对账。
  6. 标记完成（仅 已对账/已完成 状态可点）；若后续被报价转成销售合同 → 自动 已成单（列表绿色加粗）。

业务分支

分支规则
状态机守卫推送版师仅限 待派单/打样中；标记完成仅限 已对账/已完成——乱序操作被拦截并提示
工时对账（多款合并）对账管理 →「工时对账」：自由勾选多款样衣（须同一版师、每款件数×工时单价>0）合并生成一张工时对账单（类型 LABOR、币种 CNY、按月归账期），样衣转已对账
复制样衣行操作「复制」→ 基本信息+材料明细复制为新单（待派单）
删除仅「待派单」可删；被报价等引用时拦截
变更记录详情内可查看历次变更版本

数据流转：材料明细 → 报价「从样衣导入」；件数/工时单价 → 工时对账单；被成单状态 ← 报价回写。

04客户报价

对客报价核价 · 编号 Q- · 版师/打样整个模块不可见

标准流程

  1. 新建：选中间商（必填，受客户机密过滤）、可选最终买家、款号、币种/汇率/利润率。费用明细自动带 6 行常用费用。
  2. 「从样衣导入」把样衣材料明细带入报价明细行（部位/品名/门幅/颜色/供应商/耗用/损耗%/RMB单价）。
  3. 核对 RMB 合计 / USD 报价（汇率、利润率换算）→ 发出报价 → 已报价。
  4. 客户砍价 → 「客户调整」→ 客户调整 → 改完再次发出。
  5. 谈成 → 「转销售合同」→ 已成单，关联样衣同步已成单。之后去订单模块「从报价一键导入」。

业务分支

分支规则
超阈值审批发出报价时金额超系统阈值 → 转 待审批，主管「审批」通过后方可发出
编辑窗口仅 草稿/客户调整 状态可编辑（覆盖式保存）
订单导入门槛只有 已报价/客户调整/已成单 的报价才能被订单导入——草稿不可
对客打印打印报价单自动隐藏供应商名（对客脱敏）；抬头取默认本司主体
客户机密联动非授权用户在报价列表/详情中看不到机密客户的报价单（整单隐藏、详情404）
复制 / 删除复制→新草稿；仅草稿可删

05订单管理

采购与生产的共同源头 · 编号 O-YYYYMMDD-序号 · 大货总数=尺码矩阵所有格子之和

标准流程

  1. 新建订单 → 「从报价一键导入」：带出款号/客户/中间商/买家 + 报价明细复制为材料明细（单件耗用=报价耗用，快照）。业务只补：客户 PO 号、约定交期、生产工厂（下拉=委外加工商）。
  2. 填尺码数量搭配矩阵：行=款号·颜色·尺码；列=各 PO（列头三行：PO号/目的地/收货人，可「＋加 PO 列」）；格填数量。行 TOTAL、各 PO 合计、总计自动回填「大货总数量」；下方自动出「总数量搭配（不分PO）」汇总。
  3. 或用「📋 Excel 导入」四步：下载模板（CSV，Excel 直接打开填写）→ 上传 → 系统校验（款号/尺码/非负整数/PO 表头，异常行逐条提示）→ 确认入库生成矩阵。
  4. 核对材料明细/用料核算：系统采购量 = 大货总数 × 单件耗用 × (1+损耗%)；整数类单位（个/条/只…）自动向上取整（可逐行强制/关闭）；最终采购量偏离系统值超 ±10% 弹二次确认。
  5. 每行可设拆分：不拆 / 按尺码 / 按颜色——展开行实时预览分色分码出量（某色该料量=该色件数×耗用×(1+损耗)），生成材料合同时按此拆行。
  6. 保存 → 「推进状态」下单（超阈值需主管审批）→ 去合同模块生成材料/加工合同。

状态机

草稿 → 推进 → 已下单 → 生成合同自动 → 已生成合同 → 生产中 → 对账确认自动 → 已完成。仅草稿可编辑/删除。

06合同管理（材料 / 加工两类编辑页）

编号 HT-YYYYMMDD-序号，补料合同 补料-原合同号-序号 · 列表页两个新建按钮分别进两类编辑页

建材料合同（原料/辅料购销协议：甲方=供应商，乙方=本司）

  1. 合同列表 → 新建材料合同 → 编辑页。选关联订单（关联款号自动带出，可多选补充）、选甲方(供方)（自动带出住所/代表/电话）。乙方=本司主体（默认抬头）、乙方代表默认登录人可改。
  2. 「🔵 选订单款号带入」→ 货物明细自动生成：品名/规格/颜色/尺码/单位/数量=采购量(含损耗)/单价（带参考价可改）/小计/款号/交货期限（默认=订单交期−45天）/照片（可传材料图）。分色/分码材料自动按矩阵拆多行。可加行/删行/微调（不回写订单）。
  3. 选填：签约地点/日期（默认今天）、发货地址（材料发往加工厂，供门户发货带入）、担保人(丙方)+身份证照片、定金/中期/尾款比例（三者之和必须=100%）、账期（材料默认 90 天）。
  4. 合同条款分区：质量标准/交货方式/验收/货款结算/违约/争议 六条模板填空可改，随合同归档；填了担保人 → PDF 自动插入丙方连带担保条款。
  5. 保存 → 生成合同号；「📄 生成 PDF」出正式合同文本（可打印/另存 PDF）；「📤 推送供应商」进门户。

建加工合同（委托加工合同：受托方=加工厂，委托方=本司）

流程同上，差异点：甲方过滤=委外加工商；货物明细列为 货号/品名及规格/数量（带订单大货数）/加工单价（业务现填）/总价/交期（默认=订单交期−10天）；多一个「价格包含项」分区——工缴/裁剪/线/后道/运费/纸箱/拷贝纸/干燥剂/运杂费/进仓费 勾选（默认全含、胶袋不含）+ 增值税%（默认13，含税不另计）+ 其他手填，勾选汇入 PDF「以上价格包含…」段落，不改合同金额；账期默认 45 天。

业务分支

分支规则
超阈值审批推送时金额超阈值 → 转「待审批」，主管审批通过后才可推送
推送前置校验该工厂必须已开通门户账号，否则拦截提示
撤销推送供应商未盖章前可「撤销推送」回草稿修改；重推后门户红条提示「合同已更新，请重新核对」，供应商重新盖章后标记清除
编辑锁定草稿全字段可改（改明细自动重算总额；金额变化会重置已通过的审批）；已推送/盖章后仅「备注」可改——要改先撤销推送
盖章快照供应商盖章瞬间锁定金额/明细快照，此后对账按快照单价
补料（仅材料合同）「🧩 补料」= 复制原合同全部信息与材料清单，各行数量清零由业务填实际补料量；编号「补料-原合同号-NN」不占主序号，独立走门户
复制 / 删除复制→新草稿（签约日期重置为今天）；仅草稿可删；被发货/对账引用禁止删
发货批次审批合同详情「发货批次」表：业务/主管对每批发货「通过/驳回」——通过后供应商门户才能勾选该批次对账；已被对账引用的批次不可改审批
供应商拆单订单侧「生成材料合同」按材料供应商自动拆成多张合同；供应商没建工厂档案的归入「未匹配」提示补档
到期日= 最后发货日 + 账期天数；详情与应付列表逾期红标

07供应商门户（手机 H5）· 四步顺序锁定

🔖盖章 → 📦发货 → 🧾对账 → 💵开票，前一步未完成后面按钮不可点

四步操作

  1. 🔖 我要盖章：打开合同详情核对材料/金额 → 勾选「我已阅读并同意合同条款」（不勾不能提交）→ 确认盖章。盖章后明细锁定快照，进入发货。
  2. 📦 我要发货：填本次实发数量 + 快递公司/单号 + 附件（装箱单/货物照片）+ 备注 → 提交。可多次发货，页面显示累计已发；每批发货单号 FH-款号-序号，逐批锁定当时合同单价。每批提交后等达泰业务审批（批次卡显示 待审批/已审批/已驳回/已对账 标签）。
  3. 🧾 我要对账：至少一批发货被审批通过后按钮解锁 → 弹窗勾选批次（未审批/已对账的置灰不可选，默认全选可对账批次）→ 系统按 合同单价×发货数量 自动算对账金额 → 「确认对账·推业务审批」生成对账单 DZ-款号-序号（直接进业务复核）。业务确认后合同转「已对账」解锁开票。
  4. 💵 我要开票：填发票号 + 发票金额 + 上传发票（照片/PDF）→ 提交。发票金额必须与对账金额一致（允许一分钱以内误差），否则无法提交；发票号全局查重防重复报销。提交成功后系统自动生成付款申请推给达泰财务。

拦截与分支

场景系统行为
超发本次发货后累计超合同量 → 拦截提示，弹「超发确认」二次确认后才放行
勾未审批批次对账拒绝并列出未通过审批的批次号
同批次重复对账拒绝（批次被对账单占用；内部删除该对账单后自动释放可重选）
对账前开票拒绝：「开票须在对账通过后进行」
合同被撤回重推门户红条「合同已更新，请重新核对」，需重新盖章
加工合同专属详情多一块「订单明细（加工同步）」：订单号/款号/大货数/纸板/尺寸表/填充量附件/订量明细

08对账管理

编号 DZ-款号-序号（无合同 DZ-费用-序号）· 三种类型 · 业务提交 + 主管复核二级审批

三种对账单

类型怎么来金额口径
合同对账供应商门户「我要对账」勾批次自动生成（直接进待复核）；或业务在对账管理手工新建挑出货批次Σ 批次快照单价×数量；一张单只能同一类型合同（材料或加工），不可混
无合同空白对账单业务手工新建，子类型三选一：费用 / 现金无票 / 预付款Σ 费用明细行（费用名/金额/款号/附件）
工时对账勾选多款样衣（同一版师）合并生成Σ 各款件数×工时单价，CNY，按月归账期

审批流（二级）

草稿 →业务「提交复核」→ 待复核 →主管「复核确认」→ 已确认 →付款付清自动→ 已付款。主管也可「整单退回」（回草稿并记退回批注）。

关键规则

  • 发票校验：含票对账单复核确认时校验发票金额与对账金额一致（允许一分钱以内误差），不一致拦截、不进付款。
  • 发票号全局唯一：录重复发票号直接拒绝。
  • 合同对账确认 → 联动合同门户状态「已对账」+ 订单「已完成」。
  • 删除仅限草稿；删除后释放被占用的发货批次（供应商可重新勾选对账）。

09付款管理

编号 PR-（无合同 PR-NC-）· 预付款冲抵 · 审批后分批付款 · 账期到期日逾期高亮

预付款 Tab

财务/管理员创建预付款（工厂/金额/付款日期）；页面实时显示所选工厂可用预付余额。新建付款申请时可「冲抵预付款」——冲抵额不得超申请金额、不得超可用余额（均拦截）。

付款申请 Tab · 标准流程

  1. 来源两种：门户开票自动生成（自动进入待审批，带账期/到期日，备注里注明发票号）；或手工新建（类型 合同付款/非合同付款，可冲抵预付）→ 草稿 → 「提交」。
  2. 管理员「批准」（同时扣减预付余额）或「驳回」（填原因）。
  3. 已审批后点「付款」→ 财务付款弹窗（可分批）：显示 应付总额 / 已付总额 / 未付余额 三栏 + 历史分批记录表 → 填 付款方式（银行转账/承兑汇票/其他）、付款日期、本次付款金额（默认=余额，可改小分批付）、水单（点击/拖拽/Ctrl+V 粘贴截图上传）、备注 → 确认付款。
  4. 系统自动累计已付/未付；余额=0 整单转「已付款」并联动对账单已付款。已付清后「付款记录」可随时查历史批次。

10结算清单（成本利润核算）

编号 JS-款号-序号 · 成本/毛利仅财务与管理可见 · 按款号自动聚合

标准流程

  1. 新建 → 搜款号选订单（下拉「款号·订单号」可搜索）→ 系统按该款号自动聚合：总货款(含税)=已确认合同对账合计、期间费用=无合同对账合计；出货件数汇总自订单出货记录。可人工覆盖。
  2. 补录收汇口径：发票金额(USD)、实际收汇(USD)、结算汇率 → 系统算 结算金额(RMB)=收汇×汇率、美金单价、保本汇率。
  3. 费用与税：运杂费/快邮费/打样费/其它费用（可加费用明细行，各行可设税率——有票按税率换不含税，无票按含税全额）；财务及管理费=结算金额×7%；出口退税自动测算（可退税不含税采购额×退税率，回款到账后置「已到账」）。
  4. 登记回款；对账付款侧有新付款后点「刷新付款汇总」重算总货款。
  5. 核对 含退税净利（列表红/绿标注亏盈）→ 「确认」定稿 已确认。

规则：总货款只计合同对账（无合同费用单独进期间费用，不重复扣减）；仅草稿可删；版师/打样看列表时净利显示「—」。
