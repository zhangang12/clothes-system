📍 业务流位置

本稿是门户四步中「对账/开票」在内部的承接，向下到财务付款，是整条主线的终点。

⑥ 供应商协同门户 🧾 供应商「我要对账」──推送──▶ 【对账单 DZ-…】──▶ 业务审批 │ 通过 → 解锁供应商「我要开票」/ 退回 → 供应商重对账 ▼ 💵 供应商「我要开票」(传发票，金额=对账金额)──推送──▶ 自动生成【付款申请 PR-…】 │ 财务按账期(材料45天/加工30天)付款 ▼ ✅ 付款完成（主线终点）
💵 对账·开票·付款 · 业务规则
  • 🧾 对账单来源:⑥门户供应商「我要对账」推送;同供应商一次推送的已审批发货批次合并为一张对账单(可跨多合同/多款);金额 = 合同单价 × 已审批发货数量,单价按发货批次锁价(快照,后续改价不影响)。
  • ✅ 业务审批(v1.3 二级):业务员初审 + 业务主管二级审批通过才解锁供应商「我要开票」;退回可逐批 ✏️批注、随退回通知供应商重对账;一张对账单仅同一类型合同(材料 或 加工,不混含);累计实发 > 合同量时需业务弹窗确认超发后方可对账。
  • 💵 付款申请:供应商开票后系统自动生成付款申请(关联对账单/发票);发票金额 = 对账金额(严格相等,容差 ≤¥0.01),发票号全局查重防重复报销。
  • 📅 账期(v1.3 业务定):材料 = 发货日 + 90 天、加工 = 发货日 + 45 天(可人工改);到期日 = 发货日 + 账期,临近/逾期高亮。
  • 💰 财务付款:可分批付款(本次金额 / 已付总额 / 未付余额自动累计,余额=0 转「已付清」),每批传水单(支持截图粘贴);无合同付款(费用/现金无票/预付款)走审批、纳入应付;预付款登记款号/供应商,后续同款付款时提示并可勾选冲抵防重复。
  • 🔢 编号(v1.3 业务定):对账单 DZ-款号-序号 / 发货单 FH-款号-序号 / 付款申请 PR-款号-序号(多款对账单取主款号;前缀沿用 PR-);支持按款号、供应商+日期检索。
  • 👥 角色:业务员初审 + 业务主管二级审批对账;财务=只读金额、确认付款;供应商在门户侧操作。

✅ 校验 / 权限

  • 对账金额 = Σ(合同单价 × 已审批发货数量);仅已审批发货批次计入
  • 发票金额必须 = 对账金额(门户已拦截不符);付款金额默认=对账金额
  • 账期到期日 = 出货(最后发货)日 + 合同账期(材料45/加工30天)
  • 权限:业务审批对账(不可付款);财务付款(只读金额、不改对账);供应商仅门户操作
