1. 工厂资料 (Suppliers)

工厂资料是供应侧所有外部合作单位的“通讯录＋档案柜”——面料供应商、辅料供应商、委外加工厂、货代、测试公司等都登记在这里。一家厂商在这里建过档，全系统的采购、委外、付款等单据就都能直接选用它。

1.1 工厂资料 · 编辑页

用于新建 / 修改一条 工厂资料 记录的表单页

一般 etGeneral 35 字段
📌 业务规则
  • ★ v1.3：「厂商编号」改自动生成（S001 / S002 …，D2 决策）；新建时只读、不可手填
  • 保存时校验：「厂商名称」「工厂类型」两项必填
  • 「联系人」「联系人电话」两个主表字段保存时自动从「联系人」子表第一行取值回填，无需手填

⚡ 页面事件（系统自动执行）

时机系统做什么
保存前 · 联系人非空校验保存时先检查“联系人”明细表是否为空。为空则弹出提示“必须输入至少一个联系人！”，并中止本次保存。
保存前 · 联系人信息回填通过上述校验后，系统自动取“联系人”明细表第一行的姓名、电话，分别回填到主表的“联系人 / 联系人电话”两个只读字段，使主表始终展示主联系人。
列表页 · 超期未交易标红在工厂列表中，“最后交易日期”距今超过 30 天的记录，该单元格以红色显示，提醒关注长期没有往来的厂商。

2. 客户资料 (WCustomers)

客户资料是公司所有客户的“档案柜”，区分中间商与最终买家，记录客户的国别区域、价格条款、结汇方式、信用与合作等级、联系人、收货地址、开户银行、快件帐号、唛头等。报价、合同、出货、结汇等环节要选客户，都从这里取。客户资料属机密单据，系统对查看权限有专门控制。

2.1 客户资料 · 编辑页

用于新建 / 修改一条 客户资料 记录的表单页

机密 etSecret 47 字段
📌 业务规则
  • ★ v1.3：「客户编号」改自动生成（CN001 / CN002 …，D2 决策）；新建时只读、不可手填
  • 【机密单据】etSecret——非授权用户在「客户列表」「报价」「合同」中均不可见此客户
  • 「客户类型」=「最终买家」时自动显示「关联中间商」字段；=「中间商」时隐藏
  • 工具栏「批量授权机密权限」仅管理员可见，弹批量勾选员工对话框
  • 「联系人」「开户银行」「快件帐号」三张子表必备，对应出货 / 结汇 / 快件单据
  • ★ v1.3 删除规则（C3 决策）：本客户被任何下游单据（样衣 / 报价 / 合同）引用时 → <b style='color:#C04042'>阻止删除</b>，弹错「已被 N 张单据引用，无法删除」
  • ★ v1.3 修改同步（C1 决策）：本客户的名称/编号被改 → 实时同步到所有 草稿/已报价 状态的引用单据；已成单/已对账 不同步
  • ★ v1.3 永久保留（K2 决策）：客户停用后历史单据中的客户信息永久保留，不做版本管理
