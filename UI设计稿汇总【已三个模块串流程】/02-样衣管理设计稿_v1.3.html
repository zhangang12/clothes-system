📐 样衣管理 · UI 设计稿 v1.302.1 · 数据表 XYGL

样衣管理（原销样管理）是业务流的第一道单据——客户下试样意向时建档，跟踪到样衣寄出、客户反馈、对账完成整个闭环。本单据承担两个角色：① 款式工艺资料的源头（材料+尺寸等）；② 打样过程的载体（寄样→反馈→对账）。
编号规则：S-YYYYMMDD-序号（v1.3：加 S 前缀，如 S-20260527-01；当天顺序，D1 决策）　·　覆盖视图：业务视图编辑页 + 版师视图编辑页 + 列表页（浏览+检索一体）
📌 v1.3 改造（串流程业务方答复）：① 样衣编号加 S 前缀（S-YYYYMMDD-序号）② 板房样衣单价固定 CNY ③ 列表页"已成单"绿色加粗 ④ 阻止删除被引用样衣 ⑤ 复制功能加强 ⑥ 修改实时同步到引用本样衣的报价单 ⑦ 重要事件触发自动更新下游

📌 样衣管理 · 业务规则
  • 🔑 样衣管理是业务流第一环——下游单据（客户报价、材料采购、对账单、付款申请）全部由本单据派生
  • 📋 ★ v1.3 编号规则（D1）：样衣编号 S-YYYYMMDD-序号（如 S-20260527-01），自动生成；下游报价单 Q-YYYYMMDD-序号
  • 📋 主表【主要信息】原 23 字段：删除 10 个，保留 13 个 + 新增「样衣类别」(7 类多选)
  • 📋 子表【材料明细】原 21 列：删除 5 列，保留 16 列；「颜色」字段可动态加列；「品名」每款单独录入不从面料库选
  • 🔄 寄样跟踪是新增分组：材料寄出单号触发版师工作台推送；件数+单价填完触发对账单生成
  • 💰 ★ v1.3 板房样衣单价 固定 CNY（E1 决策）：版师工时单价 + 工时金额 都是人民币
  • 👥 角色分工：业务填基本信息+材料价格+寄出单号；版师填单耗+拉链长度+附件+寄回单号+件数+单价
  • 🟢 ★ v1.3 已成单状态（B1 决策）：被客户报价转销售合同后自动置 已成单，列表绿色加粗显示
  • 🚫 ★ v1.3 阻止删除被引用样衣（A6 决策）：有客户报价单引用本样衣时，删除操作弹错并禁止
  • 🔄 ★ v1.3 修改实时同步（C1 决策）：业务改样衣的客户/材料 → 实时同步到所有引用本样衣的 草稿/已报价 客户报价单；已成单状态不同步
  • 🔒 单价 / 工时金额 等敏感字段对部分员工不可见

✅ 字段校验规则

  • 必填：样衣编号 / 样衣类别 / 客户款号
  • 唯一（不可重复）：样衣编号
  • 材料明细 至少 1 行；明细的「品名」必填
  • 版师保存时：实际耗用 / 件数 / 工时单价 必填
  • 材料编号字段已删除（呼应第 1 章删面料库）—— 不再做编号校验

🔗 字段联动 / 显隐

  • 中间商编号 → 自动带中间商名称
  • 关联最终买家 → 自动带最终买家编号
  • 客户款号 → 业务直接录入（不再从面辅料开发挑款号）
  • 材料寄出单号 → 自动推送版师工作台 + 状态变更
  • 版师·件数 / 单价 → 工时金额公式生效 + 触发自动建对账单
  • 业务·权限不足者 → 单价 / 总价 / 工时金额 不可见
