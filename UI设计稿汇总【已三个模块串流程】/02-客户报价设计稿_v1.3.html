📐 客户报价 · UI 设计稿02.2 · 数据表 KHBJ · v1.3

客户报价记录的是公司向客户发出的正式报价单——业务流上承接「样衣管理」（材料 / 单耗 / 规格自动导入），下游对接「销售合同」（报价确认后转合同）。本版按业务方答复（三模块串流程问题清单 40 题）+ 4 条 UI 批注重构。
编号规则：Q-YYYYMMDD-序号（D1：加 Q 前缀，如 Q-20260528-01），自动生成，当天顺序
📌 v1.3 改造：① 加 Q 前缀 ② 报价明细删合计列+新增损耗/含损金额 ③ 费用明细删合计列+自动带 6 行 ④ 报价合计简化为 3 字段 ⑤ 移除版本控制 ⑥ 图片支持截图粘贴+拖文件 ⑦ 供应商带过来但 PDF 隐藏

📐 业务流位置

客户报价是业务流中段单据，上承样衣管理（S-...），下接销售合同。

📌 客户报价 · 业务规则（v1.3）
  • 🔑 业务流位置：基础资料 → 样衣管理 → 客户报价 → 销售合同
  • 📋 编号规则（D1）：报价单号 Q-YYYYMMDD-序号；关联样衣单号 S-YYYYMMDD-序号
  • 🔄 一轮报价（v1.0 已确认）：取消「两轮报价」双子表
  • 📥 报价从关联样衣导入：部位 / 品名 / 门幅 / 颜色 / 供应商 / 耗用全自动带入
  • 🔥 报价耗用业务可改（F2）：写入报价的独立快照，不回写样衣管理
  • 💰 含损金额公式：人民币单价 × 报价耗用 × (1 + 损耗)；损耗默认 3% 可改
  • 📑 费用明细新建自动带 6 行：加工费 / 线 / 包装 / 样衣费 / 测试费 / 运费
  • 🧾 报价合计精简：只保留 3 项（报价人民币价格 / 报价美元价格 / 备注说明）
  • 🖨 PDF 导出：供应商列默认隐藏（不对客户展示）
  • 🚫 不做版本控制（业务方已确认）：编辑直接覆盖，状态流转 草稿→已报价→（客户调整/已成单）
  • 🔒 转销售合同（G1）：主表 + 报价明细 + 费用明细 + 合计全部带过去，业务不可选行

✅ 字段校验

  • 必填：报价单号 / 询价日期 / 中间商 / 最终买家 / 客户款号 / 结汇方式 / 价格条款 / 外销员 / 利润率
  • 唯一：报价单号（Q-YYYYMMDD-序号，全局唯一）
  • 报价明细 至少 1 行；明细的「品名」「计量单位」「报价耗用」「人民币单价」「损耗」必填
  • 损耗 范围：0 ~ 100（百分比，可填 0）
  • 汇率必须 > 0

🔗 字段联动 / 公式

  • 中间商编号 → 自动带中间商名称、价格条款、结汇方式、贸易国别
  • 关联样衣 → 自动带客户款号 + 复制材料明细到报价明细子表（含供应商）
  • 外销币种切换 → 自动按「货币代码」表带出汇率
  • 人民币单价 → 美金单价（公式）→ 含损金额（公式）→ 报价合计（公式）
  • 损耗调整 → 含损金额（公式）→ 报价合计（公式）
  • 利润率调整 → 报价人民币价格/美元价格（含利润率）实时重算
