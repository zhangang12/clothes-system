结算清单 · 成本利润核算 07 · v1.0

订单履约的最末端核算:以款号为口径,把对账付款里该款全部付款汇总为总货款(含税/不含税),把船务的出货件数带入,自动算成本单价;财务录入收汇金额 + 结算汇率,自动生成结算金额 → 毛利 → 净利 → 保本汇率的毛利对比。
⑥对账付款(本款全部付款) ─┐ 总货款(含税/不含税) ├─▶ ⑦结算清单 ──▶ 成本单价 = 总货款 ÷ 出货件数 ⑨船务·出货件数 ───────┘ │ 财务录入 收汇金额(USD) × 结算汇率 = 结算金额(RMB) 结算金额 − 总货款(不含税) = 毛利 毛利 − 运杂/快邮/打样/其它 − 财务及管理费7% = 净利
💬 业务诉求(原话落地)
  • 新建结算清单,搜款号,带入所有此款相关付款汇总全部金额(来自⑥对账付款,分含税/不含税)和船务部分的出货件数(来自⑨船务),自动生成成本单价。
  • 财务部分填入收汇金额 + 汇率,自动生成毛利对比(结算金额 / 毛利 / 财务及管理费7% / 净利 / 保本汇率)。
  • 数据口径对齐真实「24918 款 结算单.xlsx」(利润表 + 对账单含税/不含税表)。

时机系统做什么
新建结算清单 · 搜款号拉取该款全部对账付款明细(已付)汇总总货款(含税/不含税);带入船务出货件数;算成本单价
该款新增/变更付款结算清单「刷新付款汇总」后总货款、成本单价、毛利对比联动重算
船务出货件数回填/修改出货件数更新 → 成本单价、保本汇率联动重算
财务录入收汇金额 + 汇率自动算结算金额 → 毛利 → 财务管理费 → 净利 → 保本汇率,生成毛利对比
净利为负列表「亏损」红标 + 利润汇总预警

✅ 校验 / 权限

  • 仅汇总已付款的对账明细;存在未付款明细时提示"该款尚有 N 笔未付,成本可能不全"
  • 出货件数为 0 / 缺失时不计算成本单价,提示"待船务出货"
  • 收汇金额、汇率必填才出毛利对比;只填数值不带单位
  • 权限:业务可建单/搜款号带入;财务录收汇与费用、出毛利;成本明细只读(改需回对账付款)
  • 一个款号一张结算清单,可重算;成本/收汇变更留痕
