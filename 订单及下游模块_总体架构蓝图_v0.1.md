## 2. 模块清单

### ④ 订单(主线核心)

- **定位**:报价生效后正式下大货订单,是材料采购与生产加工的共同源头。
- **编号**:`【建议】O-YYYYMMDD-序号`(延续 S-/Q- 前缀风格)。
- **数据来源**:从③客户报价【一键导入】。
- **字段分组**:
  1. **基础信息**:款号、单品单价、约定交期、报价件数、正反面图片(均报价带入、可改);客户 PO 号(手录);大货总数量(`【建议】=各尺码订货数之和`)。
  2. **尺码数量搭配**:尺码 × 数量;若启用分色,则为 尺码 × 颜色 的配比矩阵。**优先 Excel 批量导入**(固定模板,校验尺码编码/数值格式,异常弹窗)。
  3. **附件档案**:彩稿 / 大货尺寸表 / 大货纸板 / 包装资料,多文件上传/预览/下载/替换,与订单绑定。
  4. **材料明细**:从报价自动带入全部原料(部位/品名/门幅/颜色/供应商/单位/单件耗用),无需手工新增。
  5. **用料核算**:`单料订货量 = 大货总数 × 单件耗用 × (1+损耗率)`;损耗默认 3%,可按单料 / 整单调整;结果**可手动微调最终采购数量**;支持【分色/分码】拆分(见 3.2),可整单统一或针对个别物料单独设置。
- **下游**:材料明细 + 用料 → ⑤材料合同;款式/数量/交期 → ⑤′生产加工合同。

### ⑤ 材料合同(分支A)

- **定位**:向材料供应商采购面辅料的采购合同。
- **构成**:基础合同模板(甲乙方/通用条款,大部分固定)+ 材料明细表 + 合计。
- **操作**:新建 → 选材料供应商 → 多选款号 → 自动带入对应材料明细(按④设定的分色/分码出表)→ 录单价生成金额。
- **编号**:`【建议】HT-YYYYMMDD-序号`(合同统一前缀,类型字段区分材料/加工)。
- **下游**:推送供应商 → ⑥门户。

### ⑤′ 生产加工合同(分支B)

- **定位**:向加工厂下大货生产的加工合同。
- **操作**:新建 → 选加工厂 → 多选款号 → 自动带入款式/数量/交期 → 填单价生成总价 → 勾选价格包含项(线/税/胶袋/纸箱/运费/其他「其他手动录入」)。
- **下游**:推送工厂 → ⑥门户;并触发 ⑧大货资料/生产/⑨船务 内部跟踪。

### ⑥ 供应商协同门户(对外端)

- **定位**:面向外部供应商/加工厂的协同端,登录后只见与自己相关的全部合同。
- **登录**:`【建议】微信小程序 + H5`;账号与①工厂资料绑定;「推送供应商」= 生成专属链接 / 小程序通知。
- **按钮流(顺序锁定,前序未完成则后续禁用)**:
  1. **🔖 我要盖章**:一键带入自己上传的电子章 / 上传纸质盖章合同照片。(所有合同的第一步)
  2. **📦 我要发货**:填发货明细(数量明细 + 物流信息)+ 上传附件(包装/清单)。
  3. **🧾 我要对账**:系统按 `合同单价 × 已发货数量` 生成对账单 → 勾选确认 → 推送业务审批。
  4. **💵 我要开票**:上传 PDF 发票 + 配套对应对账明细 → 推送财务。

### ⑦ 对账 - 开票 - 付款(内部财务闭环)

- 业务审批对账单 → 通过后供应商方可开票 → 财务接收开票 → 生成付款申请 → 付款。
- **发票校验**`【建议】`:发票金额 = 对账金额(沿用②样衣「发票=对账金额」规则)。

### ⑧ 大货资料 / 生产　`【待细化】`

- 生产加工线特有:推送工厂后生成大货生产资料包,跟踪生产进度。本轮信息不足,后续补需求。

### ⑨ 船务资料　`【待细化】`

- 出货 / 物流 / 报关等,生产完成后接财务。本轮信息不足,后续补需求。

## 3. 端到端数据流(关键链路)

### 3.1 字段流转

```
③报价单 ──导入──▶ ④订单 ──────┬──▶ ⑤材料合同  ─推送─▶ ⑥门户 ─对账─▶ ⑦付款
  款号/单价              款号/单价/交期    └──▶ ⑤′加工合同 ─推送─▶ ⑥门户 ─对账─▶ ⑦付款
  件数                  PO号/尺码数量
  正反面图              附件(彩稿等)
  材料明细 ───────────▶ 材料明细 ─用料核算─▶ 材料订货量 ─▶ 材料合同明细数量
  (报价耗用=单件耗用)
```

### 3.2 用料核算公式链

- **不拆分**:`某料订货量 = 大货总数 × 单件耗用 × (1 + 损耗率)`
- **按分码(模式1)**:`某尺码该料量 = 该尺码订货数 × 单件耗用 × (1 + 损耗率)`,各尺码分别出行。
- **按分色(模式2)**:`某颜色该料量 = 该色配比数 × 单件耗用 × (1 + 损耗率)`,各颜色分别出行。
- 损耗默认 3%(单料 / 整单可调);各料合计 → 可**手动微调最终采购数量**;明细可导出用料汇总表。

### 3.3 对账金额链

`对账单金额 = Σ(合同单价 × 该次发货数量)`;多次发货累计对账;审批通过后允许开票;`付款申请金额 = 对账/发票金额`。

## 4. 贯穿性规则

### 4.1 编号体系　`【建议】`

| 单据 | 前缀 | 既有对照 |
|---|---|---|
| 订单 | `O-YYYYMMDD-序号` | 样衣 S- / 报价 Q- |
| 材料 / 加工合同 | `HT-YYYYMMDD-序号` + 类型字段 | — |
| 对账单 / 付款申请 / 发货单 | `【待定】` | — |

### 4.2 串流程三约束(沿用②③)

| 约束 | 在新线的应用 `【建议】` |
|---|---|
| 实时同步 | 上游「草稿/未推送」状态时,改动同步到下游 |
| 快照锁定 | 报价改 **不** 反向同步已生成订单(你已确认);订单改 **不** 同步已推送合同 |
| 引用禁删 | 已生成下游单据的订单/合同禁止删除(延续 C2/C3/A6) |

### 4.3 权限分层　`【建议】`

- 供应商门户:只见自己合同、自己的单价/金额,隔离的对外端。
- 财务:可见金额、只读;仓库/生产侧不可见单价。

### 4.4 按钮顺序锁定(门户)

`🔖盖章 → 📦发货 → 🧾对账 → 💵开票`,前一步未完成则后续按钮 disabled。
