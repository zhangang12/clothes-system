📦 订单 · UI 设计稿 v1.003 · 数据表 DD

订单是报价生效后正式下大货的单据，是材料采购与生产加工的共同源头：从③客户报价一键导入基础信息与材料明细，业务补 PO 号、尺码数量搭配、附件，系统按用料定额自动算出各材料采购量，再向下生成材料合同 / 加工合同。
编号规则：O-YYYYMMDD-序号（如 O-20260608-01，当天顺序自动生成）　·　来源：③客户报价 Q-YYYYMMDD-序号 一键导入　·　覆盖视图：编辑页（新建订单）+ 列表页
📌 v1.0 业务方答复落地：① 从报价单一键导入 ② 单件耗用＝单件用量 ③ 大货总数＝各尺码颜色数量之和 ④ 损耗默认 3%（单料/整单可调）⑤ 采购量微调超 ±10% 弹窗确认 ⑥ 附件增加「填充量」共 5 类 ⑦ 材料品名可手动调整（D5）⑧ 报价改不反向同步订单（快照）

📍 业务流位置

订单处在「报价 → 下单 → 采购/生产」的中枢，向上承接报价，向下分出材料采购线与生产加工线。

📦 订单 · 业务规则
  • 🔑 订单是采购与生产的共同源头：基础信息、材料明细均由③客户报价【一键导入】，业务只补 PO 号、尺码数量、附件。
  • 📋 编号规则：订单 O-YYYYMMDD-序号，自动生成；下游材料/加工合同 HT-YYYYMMDD-序号。
  • 🔢 大货总数量 ＝ 尺码数量搭配表所有格子之和（各尺码 × 各颜色），系统自动汇总（业务答复）。
  • 📐 用料核算：单料采购量 ＝ 大货总数 × 单件耗用 ×(1 + 损耗率)；单件耗用＝每件衣服用料多少（业务答复）；损耗默认 3%，可按单料或整单调整。
  • 🎯 分色 / 分码拆分：可整单统一，也可个别物料单独设定——按尺码或按颜色分别核算用量。
  • ⚠️ 系统算出的采购量可手动微调；微调幅度超过 ±10% 需弹窗二次确认（业务批注）。
  • 📎 附件 5 类：彩稿 / 大货尺寸表 / 大货纸板 / 包装资料 / 填充量（业务新增）。
  • ✏️ 材料品名沿用③报价带入的文本，可手动调整（D5：不重建料库）。
  • 🔒 快照：报价单事后修改 不 反向同步到已生成订单；订单一旦生成下游合同也走快照。

▣ 用料核算

用料核算 核算模式 + 公式 + 分色/分码拆分
整单核算模式： ● 不拆分 ○ 按尺码（分码） ○ 按颜色（分色） 整单损耗率默认 % （整单统一设置；个别物料可在材料明细「拆分」列与「损耗%」单独改）
📐 采购量公式
· 不拆分：单料采购量 = 大货总数 × 单件耗用 × (1 + 损耗率)
· 按分码：某尺码该料量 = 该尺码数量 × 单件耗用 × (1 + 损耗率)，各尺码分别出量
· 按分色：某颜色该料量 = 该颜色数量 × 单件耗用 × (1 + 损耗率)，各颜色分别出量
· 损耗默认 3%，可单料/整单调整；采购量可手动微调，超 ±10% 弹窗确认。
· 多款号同单时，分色/分码按颜色或尺码跨款号汇总（如黑色 ＝ 各款黑色数量之和）。
· ★ 整数类材料（拉链/扣子/螺纹，单位 个/条）：损耗加完后向上取整、保留整数（业务批注，如 1454×1.03＝1497.62 → 1498）。

✅ 字段校验规则

  • 必填：订单编号 / 关联报价单号 / 客户款号 / 约定交期 / 客户 PO 号
  • 尺码数量搭配 至少 1 个 PO 列 + 1 个数量格 > 0；每个 PO 需填目的地/收货人；大货总数量 = 总计 > 0
  • 材料明细 至少 1 行；行内「品名 / 单位 / 单件耗用 / 损耗%」必填
  • 损耗% 范围 0~100；单件耗用 > 0；整数类材料(个/条)采购量向上取整保留整数
  • 最终采购量偏离系统值 > ±10% → 必须弹窗确认后才保存
  • Excel 导入尺码：校验尺码编码与数值格式，异常行弹窗提示

🔗 字段联动 / 显隐

  • 关联报价单 → 一键带入基础字段 + 材料明细
  • 各 PO 数量 → TOTAL QTY/各 PO 合计/总计(总搭配) → 大货总数量 → 用料核算
  • 核算模式 = 按颜色/按尺码 → 材料「拆分」列默认随之；个别物料可单独覆盖
  • 单件耗用 / 损耗 / 大货总数 任一变化 → 系统采购量重算
  • 佣金 → 后续结算/对账金额关联；生产工厂 → 绑定订单、工厂侧可见
  • 权限不足者 → 单价 / 采购量金额 不可见
