系统定位

I9 服装制造管理系统，面向外贸服装制造企业，覆盖从样衣开发、客户报价、订单管理、合同执行、供应商协同、对账付款到最终结算的完整业务链条。系统以"快照锁定"和"流程串联"为核心设计原则，确保数据一致性与可审计性。

核心价值
  • 全流程数字化：打通样衣→报价→订单→合同→发货→对账→结算七个环节
  • 供应商协同：H5门户支持供应商自助完成盖章、发货、对账、发票上传
  • 财务精确：快照机制确保每笔费用溯源，超付拦截，多维利润分析
  • 数据不丢失：无物理删除，全状态流转，历史可查

安全设计
  • 双租户鉴权：内部员工账号（RBAC角色：业务/财务/管理员）vs 供应商门户账号（独立账号体系）
  • 接口权限：财务专属接口（付款审批、结算确认）需 FINANCE 角色，供应商接口只能访问自己关联合同
  • 数据隔离：供应商门户查询自动加 supplier_id 过滤，禁止跨供应商查询
  • 防重提交：付款申请、审批等关键操作加分布式锁（Redis SETNX）
  • 文件安全：OSS 临时签名URL，30分钟有效，禁止直接暴露存储桶

快照机制说明
核心原则：下游单据创建时，锁定上游关键数值为快照副本。后续上游修改不影响已创建的下游单据。
快照时机锁定内容锁定原因
报价→合同费用明细、单价、损耗率合同金额已确定
供应商盖章合同材料清单、单价盖章即合同生效，不可改
对账单创建发货时合同单价对账依据不可追溯修改
结算确认总货款、总成本、退税计算财务记账依据

07 结算清单 JS-编号利润分析
成本计算规则
有票（含增值税发票）：成本 = 发票金额 ÷ (1 + 实际税率)（不含税价）
无票：成本 = 付款金额全额（含税全额）
成本单价 = 总成本 ÷ 出货件数
利润双视图
视图净利公式用途
含退税利润净利 = 货款 - 成本 + 退税金额含出口退税后实际利润
不含退税利润净利 = 货款 - 成本货款到账前的毛利估算
退税计算
退税金额 = 含税面料采购额 × (出口退税率 - 增值税率差)
结算时机
  • 待结算：出货后，业务可创建草稿结算单（待收汇状态）
  • 已结算：收汇后，财务确认结算，生成毛利记录

6. 核心业务规则
财务计算规则
规则名公式/约束
含损单价含损单价 = 原单价 ÷ (1 - 损耗率%)
成本（有票）成本 = 发票金额 ÷ (1 + 税率)，即不含税价
成本（无票）成本 = 付款全额（含税）
成本单价成本单价 = 总成本 ÷ 出货件数
退税金额退税 = 含税面料采购额 × (退税率 - 增值税率差)
含退税净利净利 = 总货款 - 总成本 + 退税金额
不含退税净利净利 = 总货款 - 总成本
发票校验|发票金额 - 对账金额| ≤ ¥0.01
超付拦截申请金额 > (合同金额 - 已付金额) → 阻止提交
付款条款验证定金% + 中期% + 尾款% = 100%
账期规则
  • 材料合同：账期 = 最后发货日 + 45天
  • 加工合同：账期 = 最后发货日 + 30天
  • 账期字段在合同创建时由系统自动计算，也可手动修改
状态流转规则（不可逆）
除驳回外，所有状态只能正向流转，不可后退。已确认/已盖章状态的数据，核心字段不可修改。
单据状态流转不可逆节点
报价草稿→已发出→已确认→转合同已发出后锁定
合同草稿→已推送→已盖章→执行中→已完成已盖章后锁定
付款申请草稿→待审批→已审批→已付款已审批后锁定
结算单待结算→已结算确认结算后锁定
数据一致性规则
  • 无物理删除：所有主档（工厂/客户）和单据只能停用/作废，不能物理删除
  • 下游引用保护：有未完成下游单据的主档不可停用
  • 编号唯一性：同日期序号用Redis INCR保证并发安全
  • 金额精度：所有金额字段存储为 DECIMAL(15,4)，展示时四舍五入到2位
  • 汇率快照：对账/结算时锁定当日汇率（如有外币）
